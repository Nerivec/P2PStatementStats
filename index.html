This tool (fully local, the files you provide never leave your computer) allows you to get various statistics by simply giving it the account statements of the platforms you use. Once your files are loaded, simply click the Process button. You can also save your data for faster loading next time (only add the newer data from new statements).

What's this?

Support here is limited. Calculations are in USD equivalents as per the statements (value & date/time). Held underlying crypto-currencies are not considered (capital gain/loss). Because of this, the results ("interest earning") are somewhat more comparable to that of P2P. However, this also means it does not reflect the true values/returns for crypto platforms.

"Current Investments" files are required for these platforms for full stats support; their account statements alone do not provide enough data. Without providing these files, stats like country exposure will be "UNKNOWN" for these platforms.

Notes

Everything is processed locally on your computer; nothing leaves your computer. If you wish to do so, you can download the latest release and run it offline on your computer. You can unplug your Internet connection when you run it, if you want to make sure ;-).

If an update from one of the platforms breaks the saves from a previous version of this tool, you will get notified when you try to load them. If that happens, you will have to create a new save by loading all the statements again first.

If you encounter other save-related issues, make sure you use the same browser; different algorithms in different browsers may impact the process. Tests were done on Firefox (latest) without issues.

Running this tool on mobile browsers is not recommended, nor supported.

The length of the entire span available for a particular platform will affect the accuracy of the end results (longer is better, below 2 months is hardly comparable).

If you end up with a negative value for a platform in Shares (that you know is wrong), make sure you loaded the entire history for that platform (it can't know the previous value of your account otherwise).

More about Time-Weight Rate of Return algorithm on Wikipedia. This tool currently uses the "valued after flow" version.

Due to limited data available in some platform's statements, not all platforms support country exposure. Platforms that don't support it, will be classified as "Unknown". See "Platform Notes & Tips".

Platform Notes & Tips
Enable Multi-Load

Load multiple statements for a single plateform (one after another).

Some platforms limit the number of rows available in one statement file (like IUVO, 10000). Enabling Multi-Load allows to load the entire history by loading multiple statements that cover the entire span.

In the same manner, this also allows adding new transactions to a previously-loaded save. For example, you could download last month's statements from each of your platforms, and add it to your save (which would have the rest of your history). It avoids long wait time (and possibly timeouts) associated with generating full-history statements on platforms where you have a long history. Don't forget to grab your new save once you are done!

If you have a previous save, load it before anything else. Also, to be safe, you should always load statements from oldest to newest. The algorithm does its best to correct the order of transactions, but some statements are very poor data-wise, which makes sorting a bit harder.

Duplicate rows are removed automatically (each row is uniquely identified on load). If you want to load a new statement that might contain transactions already present in your session/save (overlapping dates), you can; it should not create any issue.

Only enable this feature whenever actually needed. Loading statements is faster when disabled; especially the larger the statements get.

If this feature is disabled, when loading a new statement, the corresponding platform's data is reset.
